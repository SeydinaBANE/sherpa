# Support

Besoin d'aide avec Sherpa ?

1. **Documentation** : commence par le [README](README.md) et l'[index docs](docs/README.md)
   (notamment [ONBOARDING](docs/ONBOARDING.md), [CONFIGURATION](docs/CONFIGURATION.md),
   [FAQ](docs/FAQ.md)).
2. **Bug** : ouvre une [issue](https://github.com/SeydinaBANE/sherpa/issues/new/choose)
   avec le modèle *Bug report*.
3. **Évolution** : ouvre une issue avec le modèle *Feature request*.
4. **Vulnérabilité** : ne pas ouvrir d'issue publique — voir [SECURITY.md](SECURITY.md).

Avant d'ouvrir une issue, vérifie qu'elle n'existe pas déjà et fournis un cas reproductible
(version, OS/Python, étapes, logs).
